## SECTION 1 — HONEST TAKE

**1D DP is the battleground of FAANG interviews.**

Not because the problems are hardest — they aren't. 1D DP problems are mostly Medium difficulty. But they appear constantly, and interviewers can instantly distinguish between candidates who truly understand state definition vs those who memorized a few formulas.

**What makes 1D DP actually hard:**

1. **Decode Ways (LC 91)** — looks like Fibonacci, but has 6 edge cases that trip up 80% of candidates
2. **Maximum Product Subarray (LC 152)** — if you only track max, you miss that min × negative → new max
3. **Delete and Earn (LC 740)** — doesn't LOOK like House Robber at all until you reduce it

The sub-skill being tested is: **can you define the right state AND handle the edge cases** for each problem?

**The key insight that unifies all 1D DP:**

Every 1D DP has a state `dp[i]` that represents "the answer for subproblem ending at or using first i elements." The word "ending at" vs "using first i" is critical:
- "Ending at i": the answer must INCLUDE element i (Maximum Subarray, LIS)
- "Using first i": the answer considers all possibilities up to i, may or may not include i (Coin Change, Word Break)

Getting this wrong leads to reading the wrong cell as the answer.

## SECTION 2 — THE 1D DP TAXONOMY

Before solving any 1D DP problem, classify it into one of these five categories:

### Category 1: Fibonacci-style (dp[i] from dp[i-1], dp[i-2])
- `dp[i] = dp[i-1] + dp[i-2]` or similar with 2-3 previous states
- Examples: Climbing Stairs, Min Cost Climbing Stairs, Tribonacci

### Category 2: Take or Skip (decision at each element)
- At each element, make a binary decision: take or skip
- `dp[i] = max(dp[i-1], dp[i-2] + nums[i])`
- Examples: House Robber, Delete and Earn, Stock problems

### Category 3: Reachability / Minimum Steps (jump forward)
- At each position, you can jump forward; track if reachable or minimum jumps
- Examples: Jump Game, Jump Game II, Minimum Cost For Tickets

### Category 4: String DP (match characters, count decodings)
- State = "can we segment/decode/match first i characters?"
- Examples: Decode Ways, Word Break, Regex Matching

### Category 5: Extend or Start Fresh (track running value)
- At each position, decide whether to extend previous result or start new
- Examples: Maximum Subarray (Kadane's), Maximum Product Subarray, Longest Turbulent Subarray

**Classifying correctly → applying the right template → solving in under 20 minutes.**

## SECTION 3 — TEMPLATE 1: FIBONACCI-STYLE DP (MIN COST CLIMBING STAIRS)

```cpp
// ─────────────────────────────────────────────────────────────────
// LC 746 — Min Cost Climbing Stairs
// You can start at step 0 or 1. To leave step i, you pay cost[i].
// Can climb 1 or 2 steps at a time. Minimize total cost to reach the top (past last step).
//
// State:    dp[i] = minimum cost to REACH step i (not pay to leave it)
// Transition: to reach step i, came from i-1 (pay cost[i-1])
//                              OR came from i-2 (pay cost[i-2])
//             dp[i] = min(dp[i-1] + cost[i-1], dp[i-2] + cost[i-2])
// Base cases: dp[0] = 0 (at start, no cost yet), dp[1] = 0
// Answer: dp[n] (n = len, the "top" past last step)
// ─────────────────────────────────────────────────────────────────

int minCostClimbingStairs(vector<int>& cost) {
    int n = cost.size();
    vector<int> dp(n + 1, 0);
    // dp[0] = dp[1] = 0 (can start from either, no cost to arrive)

    for (int i = 2; i <= n; i++) {
        dp[i] = min(dp[i-1] + cost[i-1],   // came from step i-1
                    dp[i-2] + cost[i-2]);    // came from step i-2
    }

    return dp[n];  // cost to reach the top (beyond array)
}
```

**TRACE for cost=[10,15,20]:**
```
n=3. dp = [0, 0, 0, 0]

i=2: dp[2] = min(dp[1]+cost[1], dp[0]+cost[0])
           = min(0+15, 0+10) = min(15,10) = 10
     (cheapest to reach step 2: come from step 0, pay cost[0]=10)

i=3: dp[3] = min(dp[2]+cost[2], dp[1]+cost[1])
           = min(10+20, 0+15) = min(30,15) = 15
     (cheapest to reach top: come from step 1, pay cost[1]=15)

Answer: dp[3] = 15
Path: Start at step 1 → pay 15 → reach top. Total: 15 ✓
```

**WHY dp[0]=dp[1]=0:** You CAN start at either step 0 or step 1 for free. The cost is paid when you LEAVE a step, not when you arrive. So arriving at step 0 or 1 as a starting point has zero cost.

**COMMON WRONG STATE DEFINITION:** `dp[i] = minimum cost to LEAVE step i`. This forces you to pay cost[0] or cost[1] upfront and makes the transition messier. "Minimum cost to REACH" is always cleaner.

## SECTION 4 — TEMPLATE 2: HOUSE ROBBER PATTERN (TAKE OR SKIP)

*Already covered in Pattern 16. This section focuses on the VARIATIONS that trip people up.*

### Variation A: Delete and Earn (LC 740) — Hidden House Robber

```cpp
// LC 740 — Delete and Earn
// When you pick num[i], you must delete ALL occurrences of num[i]-1 and num[i]+1.
// Maximize points.
//
// KEY INSIGHT: If you "take" value v, you get v * count(v) points,
// but you can't "take" v-1 or v+1.
// → This is EXACTLY House Robber on the value array!
//
// Transform: build earn[v] = v * count(v) for each value v
// Then: dp[v] = max(dp[v-1], dp[v-2] + earn[v])

int deleteAndEarn(vector<int>& nums) {
    int maxVal = *max_element(nums.begin(), nums.end());
    vector<int> earn(maxVal + 1, 0);

    // Build earn array: earn[v] = v * (frequency of v)
    for (int num : nums) earn[num] += num;

    // Now apply House Robber on earn[]
    if (maxVal == 0) return 0;
    if (maxVal == 1) return earn[1];

    int prev2 = earn[0], prev1 = earn[1];
    for (int v = 2; v <= maxVal; v++) {
        int curr = max(prev1, prev2 + earn[v]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

**THE REDUCTION EXPLAINED:**

Why does "delete adjacent values" become "can't rob adjacent houses"?

If you take value 3: earn = 3 × count(3). You can't take 2 (deleted) or 4 (deleted).
This is: "rob house 3 for earn[3] points → can't rob house 2 or house 4."

Exactly House Robber! The value domain [0..maxVal] is the "array of houses." Each "house" v has value earn[v]. Adjacent houses can't both be robbed.

**TRACE for nums=[3,4,2]:**
```
earn = [0, 0, 2, 3, 4]  (earn[2]=2, earn[3]=3, earn[4]=4)

House Robber on [0,0,2,3,4]:
prev2=0 (earn[0]), prev1=0 (earn[1])
v=2: curr = max(0, 0+2) = 2. prev2=0, prev1=2
v=3: curr = max(2, 0+3) = 3. prev2=2, prev1=3
v=4: curr = max(3, 2+4) = max(3,6) = 6. prev2=3, prev1=6

Answer: 6 (take values 2 and 4: 2+4=6)
```

### Variation B: Best Time to Buy and Sell Stock (LC 121) — Track Running Min

```cpp
// LC 121 — One transaction allowed
// State: NOT a typical dp[i] — just track the running minimum
// dp[i] = max profit selling on day i
//       = prices[i] - min(prices[0..i-1])

int maxProfit(vector<int>& prices) {
    int minPrice = INT_MAX, maxProfit = 0;
    for (int price : prices) {
        minPrice = min(minPrice, price);
        maxProfit = max(maxProfit, price - minPrice);
    }
    return maxProfit;
}
```

**WHY this is technically 1D DP:**
`dp[i] = max(dp[i-1], prices[i] - minSoFar[i])`
where `minSoFar[i] = min(prices[0..i])`.

It's space-optimized to track only `minPrice` and `maxProfit`. The DP state "max profit up to day i" only depends on the running minimum seen so far.

## SECTION 5 — TEMPLATE 3: DECODE WAYS (STRING DP WITH EDGE CASES)

This is the most commonly failed 1D DP problem in interviews. The logic seems simple but has multiple edge cases.

```cpp
// LC 91 — Decode Ways
// '1' maps to 'A', ..., '26' maps to 'Z'
// Count the number of ways to decode a digit string.
//
// State:    dp[i] = number of ways to decode first i characters (1-indexed!)
// Transition:
//   Single digit decode (s[i-1]):  if s[i-1] != '0', dp[i] += dp[i-1]
//   Double digit decode (s[i-2..i-1]): if s[i-2..i-1] in [10..26], dp[i] += dp[i-2]
// Base cases: dp[0] = 1 (empty string — 1 way to decode nothing)
//             dp[1] = 1 if s[0] != '0', else 0
// Answer: dp[n]

int numDecodings(string s) {
    int n = s.size();
    vector<int> dp(n + 1, 0);
    dp[0] = 1;  // empty string: 1 way
    dp[1] = (s[0] != '0') ? 1 : 0;  // single digit

    for (int i = 2; i <= n; i++) {
        // Single digit: s[i-1]
        // Valid if it's not '0' (since '0' alone can't be decoded)
        if (s[i-1] != '0') {
            dp[i] += dp[i-1];
        }

        // Double digit: s[i-2]s[i-1]
        // Valid if it forms a number in [10..26]
        // s[i-2] must be '1' OR (s[i-2]=='2' AND s[i-1] <= '6')
        int twoDigit = stoi(s.substr(i-2, 2));
        if (twoDigit >= 10 && twoDigit <= 26) {
            dp[i] += dp[i-2];
        }
    }

    return dp[n];
}
```

**THE SIX EDGE CASES — trace them all:**

```
Case 1: "12"
dp[0]=1, dp[1]=1 (s[0]='1', valid)
i=2: single '2' → dp[2] += dp[1] = 1
     double "12" ∈ [10,26] → dp[2] += dp[0] = 1+1 = 2
Answer: 2 (ways: "AB" or "L")

Case 2: "226"
dp[0]=1, dp[1]=1
i=2: single '2' → dp[2]+=dp[1]=1; double "22" ∈ [10,26] → dp[2]+=dp[0]=2
dp[2]=2
i=3: single '6' → dp[3]+=dp[2]=2; double "26" ∈ [10,26] → dp[3]+=dp[1]=3
dp[3]=3
Answer: 3 (ways: "BBF","BZ","VF")

Case 3: "06" — starts with 0
dp[1] = (s[0]=='0') ? 0 : 1 → dp[1]=0
i=2: single '6' → dp[2]+=dp[1]=0; double "06" NOT in [10,26] → no add
Answer: 0 (correct: "06" can't be decoded)

Case 4: "30" — double zero trap
dp[0]=1, dp[1]=1 (s[0]='3' valid)
i=2: single '0' → s[1]='0', so NO add (0 alone invalid)
     double "30" NOT in [10,26] → no add
dp[2]=0
Answer: 0 (correct: '3' would need '0' after but '0' can't stand alone)

Case 5: "10"
dp[1]=1
i=2: single '0' → s[1]='0', so NO add
     double "10" ∈ [10,26] → dp[2]+=dp[0]=1
Answer: 1 (only way: "J")

Case 6: "27"
dp[1]=1
i=2: single '7' → dp[2]+=dp[1]=1
     double "27" NOT in [10,26] (>26) → no add
dp[2]=1
Answer: 1 (only way: "BG")
```

**OPTIMIZATION — avoid `substr` and `stoi`:**

```cpp
// More efficient: compute two-digit value directly
int twoDigit = (s[i-2] - '0') * 10 + (s[i-1] - '0');
if (twoDigit >= 10 && twoDigit <= 26) dp[i] += dp[i-2];
```

**SPACE OPTIMIZED:**
```cpp
int prev2 = 1, prev1 = (s[0] != '0') ? 1 : 0;
for (int i = 2; i <= n; i++) {
    int curr = 0;
    if (s[i-1] != '0') curr += prev1;
    int twoDigit = (s[i-2]-'0')*10 + (s[i-1]-'0');
    if (twoDigit >= 10 && twoDigit <= 26) curr += prev2;
    prev2 = prev1;
    prev1 = curr;
}
return prev1;
```

## SECTION 6 — TEMPLATE 4: JUMP GAME FAMILY (REACHABILITY DP)

### Jump Game I — Can you reach the last index?

```cpp
// LC 55 — Jump Game
// At each index i, you can jump up to nums[i] steps forward.
// Can you reach the last index?
//
// Greedy approach (O(n)):
// Track the farthest reachable index at each step.
// If current index > farthest reachable → stuck, return false.

bool canJump(vector<int>& nums) {
    int maxReach = 0;
    for (int i = 0; i < nums.size(); i++) {
        if (i > maxReach) return false;         // can't reach index i
        maxReach = max(maxReach, i + nums[i]);  // extend reach from here
    }
    return true;
}
```

**DP approach (also O(n), less efficient but shows the DP structure):**

```cpp
bool canJump(vector<int>& nums) {
    int n = nums.size();
    vector<bool> dp(n, false);
    dp[0] = true;  // can always reach index 0

    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (dp[j] && j + nums[j] >= i) {  // can reach i from j
                dp[i] = true;
                break;
            }
        }
    }

    return dp[n-1];
}
// O(n²) — use greedy in practice
```

**TRACE for [2,3,1,1,4]:**
```
maxReach = 0
i=0: i(0) <= maxReach(0). maxReach = max(0, 0+2) = 2
i=1: i(1) <= maxReach(2). maxReach = max(2, 1+3) = 4
i=2: i(2) <= maxReach(4). maxReach = max(4, 2+1) = 4
i=3: i(3) <= maxReach(4). maxReach = max(4, 3+1) = 4
i=4: i(4) <= maxReach(4). maxReach = max(4, 4+4) = 8
Return true ✓
```

**TRACE for [3,2,1,0,4] (impossible):**
```
maxReach = 0
i=0: maxReach = max(0, 0+3) = 3
i=1: maxReach = max(3, 1+2) = 3
i=2: maxReach = max(3, 2+1) = 3
i=3: maxReach = max(3, 3+0) = 3
i=4: i(4) > maxReach(3) → return false ✓
```

### Jump Game II — Minimum jumps to reach last index

```cpp
// LC 45 — Jump Game II
// Greedy: at each "level" (group of reachable indices from previous jumps),
// find the farthest point we can reach. When we exhaust the current level, jump.

int jump(vector<int>& nums) {
    int jumps = 0, curEnd = 0, farthest = 0;
    for (int i = 0; i < nums.size() - 1; i++) {  // don't need to jump from last
        farthest = max(farthest, i + nums[i]);
        if (i == curEnd) {  // exhausted current "level"
            jumps++;
            curEnd = farthest;  // advance to next level
        }
    }
    return jumps;
}
```

**TRACE for [2,3,1,1,4]:**
```
jumps=0, curEnd=0, farthest=0

i=0: farthest=max(0,0+2)=2. i(0)==curEnd(0) → jumps=1, curEnd=2
i=1: farthest=max(2,1+3)=4. i(1)≠curEnd(2)
i=2: farthest=max(4,2+1)=4. i(2)==curEnd(2) → jumps=2, curEnd=4
i=3: farthest=max(4,3+1)=4. (loop ends at i < n-1 = 4, so i=3 is last)

Answer: 2 jumps ✓ (0→1→4 using nums=[2,3,1,1,4])
```

**WHY `i < nums.size() - 1`?** If we're already AT the last index, we don't need another jump. The loop `i < n-1` prevents counting an unnecessary jump from the last position.

**DP approach for minimum jumps (O(n²) — understand but use greedy):**

```cpp
// dp[i] = minimum jumps to reach index i
vector<int> dp(n, INT_MAX);
dp[0] = 0;
for (int i = 1; i < n; i++) {
    for (int j = 0; j < i; j++) {
        if (dp[j] != INT_MAX && j + nums[j] >= i) {
            dp[i] = min(dp[i], dp[j] + 1);
        }
    }
}
return dp[n-1];
```

## SECTION 7 — TEMPLATE 5: MINIMUM COST FOR TICKETS (MULTI-CHOICE DP)

This problem is a DP on calendar days where multiple durations of passes are available.

```cpp
// LC 983 — Minimum Cost For Tickets
// Travel on certain days. Can buy 1-day, 7-day, or 30-day passes.
// Minimize total cost.
//
// State: dp[i] = minimum cost to travel on all needed days in days[0..?] 
//                up through day i
// BUT: not all days[i] are travel days — only use passes when needed.
//
// Better state: dp[i] = min cost to handle all travel days from day i onwards
//               (suffix DP) — OR index into sorted travel days.
//
// Alternative: dp[i] = min cost to cover all travel days up to day i
//              where i ranges over actual calendar days 1..365

int mincostTickets(vector<int>& days, vector<int>& costs) {
    int lastDay = days.back();
    set<int> travelDays(days.begin(), days.end());

    vector<int> dp(lastDay + 1, 0);
    // dp[i] = minimum cost to handle all travel up to day i

    for (int i = 1; i <= lastDay; i++) {
        if (travelDays.find(i) == travelDays.end()) {
            // Not a travel day — cost same as yesterday
            dp[i] = dp[i-1];
            continue;
        }

        // It IS a travel day — try all three pass options
        int opt1 = dp[i-1] + costs[0];              // 1-day pass
        int opt7 = dp[max(0, i-7)] + costs[1];      // 7-day pass (covers days i-6 to i)
        int opt30 = dp[max(0, i-30)] + costs[2];    // 30-day pass (covers days i-29 to i)

        dp[i] = min({opt1, opt7, opt30});
    }

    return dp[lastDay];
}
```

**TRACE for days=[1,4,6,7,8,20], costs=[2,7,15]:**
```
lastDay=20
travelDays={1,4,6,7,8,20}

dp[0]=0

i=1: travel day. opt1=dp[0]+2=2, opt7=dp[max(0,-6)]+7=dp[0]+7=7, opt30=0+15=15
     dp[1] = min(2,7,15) = 2

i=2: NOT travel. dp[2] = dp[1] = 2
i=3: NOT travel. dp[3] = 2

i=4: travel day. opt1=dp[3]+2=4, opt7=dp[max(0,-3)]+7=dp[0]+7=7, opt30=0+15=15
     dp[4] = 4

i=5: NOT travel. dp[5]=4

i=6: travel. opt1=dp[5]+2=6, opt7=dp[max(0,-1)]+7=dp[0]+7=7, opt30=0+15=15
     dp[6]=6

i=7: travel. opt1=dp[6]+2=8, opt7=dp[max(0,0)]+7=dp[0]+7=7, opt30=0+15=15
     dp[7]=7

i=8: travel. opt1=dp[7]+2=9, opt7=dp[1]+7=2+7=9, opt30=0+15=15
     dp[8]=9

i=9..19: NOT travel. dp[9..19]=9

i=20: travel. opt1=dp[19]+2=11, opt7=dp[13]+7=9+7=16, opt30=dp[max(0,-10)]+15=0+15=15
      dp[20] = min(11,16,15) = 11

Answer: 11
```

**WHY `dp[max(0, i-7)]`?** If `i < 7`, then `i-7` is negative — clamp to 0. Using a 7-day pass that covers day `i` means we've already paid for all travel from day `i-6` to `i`. So we only need the cost up to day `i-7` (the day before the pass starts).

## SECTION 8 — TEMPLATE 6: MAXIMUM PRODUCT SUBARRAY

The classic extension of Kadane's — must track BOTH maximum AND minimum at each step.

```cpp
// LC 152 — Maximum Product Subarray
// Key insight: a large negative × a negative = a large positive
// So the current minimum (most negative) might become the next maximum!
//
// State: at each index i, track:
//   maxDP[i] = maximum product of any subarray ENDING at i
//   minDP[i] = minimum product of any subarray ENDING at i
//
// Transition:
//   If nums[i] > 0: maxDP[i] = max(nums[i], maxDP[i-1]*nums[i])
//                   minDP[i] = min(nums[i], minDP[i-1]*nums[i])
//   If nums[i] < 0: maxDP[i] = max(nums[i], minDP[i-1]*nums[i])  ← min becomes max!
//                   minDP[i] = min(nums[i], maxDP[i-1]*nums[i])  ← max becomes min!
//   If nums[i] = 0: reset both to 0

int maxProduct(vector<int>& nums) {
    int maxProd = nums[0], minProd = nums[0], result = nums[0];

    for (int i = 1; i < nums.size(); i++) {
        // When nums[i] is negative, max and min SWAP roles
        if (nums[i] < 0) swap(maxProd, minProd);

        maxProd = max(nums[i], maxProd * nums[i]);   // extend or start fresh
        minProd = min(nums[i], minProd * nums[i]);   // extend or start fresh

        result = max(result, maxProd);
    }

    return result;
}
```

**TRACE for [2,3,-2,4]:**
```
maxProd=2, minProd=2, result=2

i=1: nums[1]=3>0. No swap.
  maxProd = max(3, 2*3) = max(3,6) = 6
  minProd = min(3, 2*3) = min(3,6) = 3
  result = max(2, 6) = 6

i=2: nums[2]=-2<0. SWAP: maxProd=3, minProd=6
  maxProd = max(-2, 3*(-2)) = max(-2,-6) = -2
  minProd = min(-2, 6*(-2)) = min(-2,-12) = -12
  result = max(6, -2) = 6

i=3: nums[3]=4>0. No swap.
  maxProd = max(4, -2*4) = max(4,-8) = 4
  minProd = min(4, -12*4) = min(4,-48) = -48
  result = max(6, 4) = 6

Answer: 6 (subarray [2,3])
```

**TRACE for [2,-5,-2,-4,3]:**
```
maxProd=2, minProd=2, result=2

i=1: -5<0. SWAP: max=2,min=2
  maxProd = max(-5, 2*(-5)) = max(-5,-10) = -5
  minProd = min(-5, 2*(-5)) = min(-5,-10) = -10
  result = max(2,-5) = 2

i=2: -2<0. SWAP: max=-10,min=-5
  maxProd = max(-2, -10*(-2)) = max(-2,20) = 20
  minProd = min(-2, -5*(-2)) = min(-2,10) = -2
  result = max(2,20) = 20

i=3: -4<0. SWAP: max=-2,min=20
  maxProd = max(-4, -2*(-4)) = max(-4,8) = 8
  minProd = min(-4, 20*(-4)) = min(-4,-80) = -80
  result = max(20,8) = 20

i=4: 3>0.
  maxProd = max(3, 8*3) = 24
  minProd = min(3, -80*3) = -240
  result = max(20,24) = 24

Answer: 24 (subarray [-5,-2,-4,3] = 120... wait, let me recheck)
Actually [-5,-2,-4,3]: (-5)*(-2)*(-4)*(3) = -120 (negative).
[-2,-4,3]: (-2)*(-4)*(3) = 24. Correct!
```

**THE SWAP TRICK EXPLAINED:**

When `nums[i] < 0`:
- `oldMax × nums[i]` = (big positive) × (negative) = big NEGATIVE → contributes to NEW min
- `oldMin × nums[i]` = (big negative) × (negative) = big POSITIVE → contributes to NEW max

By swapping before computing, `maxProd` contains the old min and `minProd` contains the old max, which are then multiplied by `nums[i]` correctly.

## SECTION 9 — TEMPLATE 7: LONGEST TURBULENT SUBARRAY

```cpp
// LC 978 — Longest Turbulent Subarray
// Subarray is turbulent if:
// arr[i-1] < arr[i] > arr[i+1]  (peak) OR
// arr[i-1] > arr[i] < arr[i+1]  (valley)
// — alternating comparisons
//
// State:
//   inc[i] = length of longest turbulent subarray ENDING at i,
//             where arr[i-1] < arr[i] (last comparison was <)
//   dec[i] = length of longest turbulent subarray ENDING at i,
//             where arr[i-1] > arr[i] (last comparison was >)
//
// Transition:
//   If arr[i] > arr[i-1]: inc[i] = dec[i-1] + 1  (came from a valley)
//                          dec[i] = 1              (just arr[i] alone)
//   If arr[i] < arr[i-1]: dec[i] = inc[i-1] + 1  (came from a peak)
//                          inc[i] = 1
//   If arr[i] == arr[i-1]: inc[i] = dec[i] = 1   (break the streak)

int maxTurbulenceSize(vector<int>& arr) {
    int n = arr.size();
    if (n == 1) return 1;

    int inc = 1, dec = 1, result = 1;

    for (int i = 1; i < n; i++) {
        if (arr[i] > arr[i-1]) {
            inc = dec + 1;
            dec = 1;
        } else if (arr[i] < arr[i-1]) {
            dec = inc + 1;
            inc = 1;
        } else {
            inc = 1;
            dec = 1;
        }
        result = max(result, max(inc, dec));
    }

    return result;
}
```

**TRACE for [9,4,2,10,7,8,8,1,9]:**
```
i=1: arr[1]=4<arr[0]=9. dec=inc+1=2, inc=1. result=2
i=2: arr[2]=2<arr[1]=4. dec=inc+1=2, inc=1. result=2
i=3: arr[3]=10>arr[2]=2. inc=dec+1=3, dec=1. result=3
i=4: arr[4]=7<arr[3]=10. dec=inc+1=4, inc=1. result=4
i=5: arr[5]=8>arr[4]=7. inc=dec+1=5, dec=1. result=5
i=6: arr[6]=8==arr[5]=8. inc=1, dec=1. result=5
i=7: arr[7]=1<arr[6]=8. dec=inc+1=2, inc=1. result=5
i=8: arr[8]=9>arr[7]=1. inc=dec+1=3, dec=1. result=5

Answer: 5 (subarray [4,2,10,7,8] — alternates: <, >, <, >)
```

**WHY two states `inc` and `dec`?** The turbulence rule cares about the DIRECTION of the last comparison. `inc` tracks streaks ending with an upward step; `dec` tracks streaks ending with a downward step. A new upward step can only extend a streak ending with a downward step (alternating is required).

## SECTION 10 — TEMPLATE 8: WORD BREAK (COMPLETE VERSION)

*Covered in Pattern 16. This section adds the important extension: Word Break II (return all segmentations).*

```cpp
// LC 140 — Word Break II
// Return all possible sentences from the segmentation
// Use top-down DP with memoization + backtracking

class Solution {
    unordered_map<int, vector<string>> memo;  // start → list of sentences

    vector<string> solve(const string& s, unordered_set<string>& dict, int start) {
        if (memo.count(start)) return memo[start];

        vector<string> result;
        if (start == s.size()) {
            result.push_back("");  // base case: empty string after full segmentation
            return result;
        }

        for (int end = start + 1; end <= s.size(); end++) {
            string word = s.substr(start, end - start);
            if (dict.count(word)) {
                vector<string> rest = solve(s, dict, end);
                for (const string& sentence : rest) {
                    result.push_back(word + (sentence.empty() ? "" : " " + sentence));
                }
            }
        }

        return memo[start] = result;
    }

public:
    vector<string> wordBreak(string s, vector<string>& wordDict) {
        unordered_set<string> dict(wordDict.begin(), wordDict.end());
        return solve(s, dict, 0);
    }
};
```

**WHY memoization is critical for Word Break II:** Without it, overlapping suffixes are recomputed exponentially. Example: if "catsanddog" can be split as "cat|sand|dog" and "cats|and|dog", both eventually call `solve("dog")`. Memoization ensures this is computed once.

**Interviewer:** "A message encoded as digits maps to letters: 1→A, 2→B, ..., 26→Z. Count the number of ways to decode a given digit string."

**Candidate:**

*[First minute — identify state and transitions]*
> "This is string DP. State: dp[i] = number of ways to decode the first i characters. Each character can be decoded as a single digit (if valid: '1'-'9') or as part of a two-digit number (if valid: '10'-'26'). I'll use 1-indexed dp with dp[0]=1 as base case for the empty string."

> "The tricky part: '0' cannot be a single-digit decode (no letter maps to 0). And two-digit numbers only go up to 26 — '27', '30', etc. are invalid. Also '06' is not a valid two-digit decode because it's not in [10..26]."

*[Code — 5 minutes]*
```cpp
int numDecodings(string s) {
    int n = s.size();
    vector<int> dp(n + 1, 0);
    dp[0] = 1;
    dp[1] = (s[0] != '0') ? 1 : 0;
    for (int i = 2; i <= n; i++) {
        if (s[i-1] != '0') dp[i] += dp[i-1];
        int two = (s[i-2]-'0')*10 + (s[i-1]-'0');
        if (two >= 10 && two <= 26) dp[i] += dp[i-2];
    }
    return dp[n];
}
```

### Interview Simulation 2: LC 45 — Jump Game II

**Interviewer:** "Given jump lengths, find the minimum number of jumps to reach the last index."

**Candidate:**

*[First 30 seconds — classify and choose approach]*
> "This is reachability DP. I can solve it greedily in O(n): at each 'step', I explore all positions reachable in the current jump count, track the farthest position reachable from any of them, then take one jump to advance to that farthest position."

*[State the algorithm clearly]*
> "I maintain: jumps (count), curEnd (rightmost position reachable with current jumps), farthest (rightmost reachable by taking one more jump). When I reach curEnd, I increment jumps and update curEnd to farthest. I loop up to index n-2 because if I'm already at n-1, I'm done."

*[Code — 3 minutes]*
```cpp
int jump(vector<int>& nums) {
    int jumps = 0, curEnd = 0, farthest = 0;
    for (int i = 0; i < (int)nums.size() - 1; i++) {
        farthest = max(farthest, i + nums[i]);
        if (i == curEnd) { jumps++; curEnd = farthest; }
    }
    return jumps;
}
```

*[Follow-up]: "Prove this greedy is optimal."*
> "At each jump, we're choosing the farthest reachable position. Any other choice reaches strictly fewer positions — it can only delay reaching the destination. Formally: if the greedy takes k jumps, any alternative must take ≥ k jumps. Proof by induction on k: the greedy's first jump reaches the farthest possible position from the start. Any optimal solution's first jump reaches at most this far. So after the first jump, the greedy has at least as many reachable positions as any optimal solution."

**RED FLAGS:**
- Looping to `nums.size()` instead of `nums.size() - 1` (off-by-one causes extra jump)
- Using the O(n²) DP instead of greedy (correct but unnecessary complexity)
- Not being able to explain why greedy works

### Interview Simulation 3: LC 740 — Delete and Earn

```cpp
int deleteAndEarn(vector<int>& nums) {
```

## SECTION 17 — MENTAL MODEL CHECKPOINT

Seven questions. Answer before reading.

---

**Q1: In Min Cost Climbing Stairs, the state is "dp[i] = min cost to REACH step i." Why is this better than "min cost to LEAVE step i"?**

> "Min cost to REACH step i" gives the cleaner base cases: dp[0] = dp[1] = 0 (you can start at either step for free, no cost incurred yet).
>
> "Min cost to LEAVE step i" would require dp[0] = cost[0], dp[1] = cost[1] as base cases, which are correct but make the transition `dp[i+1] = min(dp[i-1] + cost[i-1], dp[i] + cost[i])` slightly less intuitive.
>
> The "reach" definition also naturally handles the "top" — `dp[n]` represents reaching the top (beyond the last step) without ever having to "leave" it, so we don't accidentally add a final step cost.

---

**Q2: Decode Ways has TWO base cases: dp[0]=1 and dp[1]. Explain WHY both are necessary.**

> `dp[0] = 1`: The empty prefix has exactly one decoding — the empty sequence. This is needed for the two-digit decode transition: when `dp[i] += dp[i-2]` at `i=2`, it adds `dp[0]`. Without `dp[0]=1`, a two-character string like "12" would not count the "L" (one-letter) decoding.
>
> `dp[1] = 1 if s[0]!='0' else 0`: The single-character prefix has one decoding if the character is valid ('1'-'9'), zero if it's '0'. This handles the edge case: "06" — since s[0]='0', dp[1]=0, and dp[2] for the two-digit "06" check also returns 0 (06 is not in [10..26]). Result: 0 (correct).

---

**Q3: Maximum Product Subarray tracks BOTH max and min at each step. Why isn't tracking just the max sufficient?**

> Example: nums = [-3, -5, 2]. After processing -3 and -5:
> - `maxProd = 15` (product of two negatives → positive)
> - At -3 only: maxProd = -3, minProd = -3
> - At -5: if we only track max... maxProd = max(-5, -3×-5) = max(-5,15) = 15. We'd lose track of -3 as a potential "running minimum" that could combine with a future negative.
>
> If the next number were -6: min(-3) × (-6) = 18 would be the new maximum. Without tracking minProd, we couldn't compute this.
>
> The minimum (most negative product) is the "stored negative energy" that a future negative number can flip into the maximum.

---

**Q4: In Jump Game II, why does the loop condition `i < nums.size() - 1` (not `i < nums.size()`) prevent an extra jump?**

> We're asking: how many jumps does it take to REACH the last index? We don't need to jump FROM the last index (it's our destination).
>
> If the loop went to `i = n-1`: when `i == curEnd` at the last position, we'd increment `jumps` unnecessarily — as if we need one more jump FROM the last position to get "past" it.
>
> By stopping at `n-2`, we ensure the last `jumps++` happens at the last "interior" step that makes the destination reachable, not at the destination itself.

---

**Q5: Delete and Earn reduces to House Robber. What is the formal invariant that makes this reduction valid?**

> House Robber invariant: "can't rob two adjacent houses." This means: selecting house v prevents selecting house v-1 and v+1.
>
> Delete and Earn invariant: "selecting value v deletes all occurrences of v-1 and v+1, making them unchoosable."
>
> The reduction: if we build earn[v] = v × count(v), then "choosing to earn from value v" is equivalent to "robbing house v for earn[v] points" with the constraint that "houses v-1 and v+1 can't be robbed."
>
> The key: the earn values are indexed by the value domain (0 to maxVal), not by array position. Adjacent indices in the earn array correspond to adjacent integer values — exactly the House Robber adjacency constraint.

---

**Q6: Word Break uses dp[0]=true. What does "the empty prefix can be segmented" mean, and why is it needed?**

> It means: the string of length 0 (before the first character) is trivially segmentable — using zero words. This is the empty segmentation.
>
> Without `dp[0]=true`, when checking if the entire string (up to index n) can start with a word from index 0: `if (dp[0] && dict.contains(s[0..k-1]))`, dp[0] must be true. Otherwise, no word starting at position 0 would be considered valid.
>
> `dp[0]=true` is the "empty product" convention in DP: the base case for zero items, which always exists as a valid trivial case.

---

**Q7: Why does the Longest Turbulent Subarray need two DP states (inc, dec) instead of just one?**

> One state can't distinguish between "the last step went up" and "the last step went down." But the turbulence rule requires ALTERNATING directions — an upward step must follow a downward step, and vice versa.
>
> If we only tracked `length = length of longest turbulent subarray ending at i`, we couldn't compute the next value: `length` of the subarray ending at `i+1` depends on whether `arr[i] > arr[i-1]` (it can extend a subarray ending with a downward step) or `arr[i] < arr[i-1]` (it can extend a subarray ending with an upward step).
>
> With two states:
> - `inc` = length of turbulent subarray ending at i with last step being upward
> - `dec` = length of turbulent subarray ending at i with last step being downward
>
> `inc[i]` can only come from `dec[i-1]` (alternating). `dec[i]` can only come from `inc[i-1]`. This captures the alternating pattern precisely.

```cpp
// Then apply House Robber on earn[]

// Binary choice per element → Take or skip

// Count ways to decode string → String DP (1-indexed, dp[0]=1)
// Track product with negatives → Min AND max both
// Hidden adjacency "can't pick adjacent values" → House Robber reduction
```
